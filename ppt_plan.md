# MindTune PPT 세부 Plan (최종)

## 결정 사항
- 시간: 15분 / 슬라이드당 ~1분
- 컬러: 네이비(#1E3A5F) + 틸(#4DA8A0) + 코랄(#E8735A)
- 보조: 본문(#2D3748), 서브(#718096), 배경(#FFFFFF), 블록(#F7FAFC)
- 다이어그램: python-pptx 도형으로 직접 생성
- 청중: 기술 면접관 (AI/ML 배경)
- 캡처: 플레이스홀더

## 슬라이드 세부 구성 (15장)

### 0. 목차 (1장)
- 전체 14개 섹션 목차 표시
- 틸 넘버링 + 네이비 텍스트

### 1. 컨셉 및 비즈니스 목표 (1장)
- 좌: "MindTune = 음악 심리 케어 에이전트"
- 핵심 3줄: 회사 도메인(의료AI+에이전틱AI), Mental Health Label 데이터셋, 음악치료 임상 근거
- 우: 키워드 박스 3개 (의료AI / 에이전틱AI / 음악치료)

### 2. 데이터셋 설명 (1장)
- 좌: 32,833곡 25개 컬럼, 전처리→26,229곡
- 우: 데이터셋 구조 표 (메타데이터/장르/오디오피처/MH라벨)
- 하단: 6개 장르 분포 바차트 (python-pptx 도형)

### 3. 데이터셋 EDA (1장)
- 좌: 핵심 발견 4가지 (MH 불균형, 감정-피처 상관, 장르 프로파일, 유사곡 시뮬레이션)
- 우: [EDA 캡처 플레이스홀더] 레이더차트/산점도
- 화살표: 각 발견 → 설계 의사결정 연결

### 4. 기술 스택 (1장)
- 아이콘 그리드 레이아웃 (2행 4열)
- Python / Streamlit / OpenAI API / ChromaDB+BM25 / Spotify / SQLite / Plotly / Docker
- 각 아이콘 = 틸 원형 + 네이비 텍스트

### 5. 아키텍처 구조 (1장)
- 중앙: Gate → Music Agent → Care + Spotify 파이프라인 다이어그램
- python-pptx 도형: 사각형(에이전트) + 화살표 + 병렬 분기
- 색상: Gate=틸, Music=네이비, Care=코랄, Spotify=틸

### 6. 에이전트 구조 (1장)
- 3단 구성 (Gate | Music | Care)
- Gate: 키워드→LLM→체이닝 3단계
- Music: 6개 도구 목록 + ReAct 루프 + Thinking Mode
- Care: intent별 강도 + 가드레일

### 7. 데이터 전처리 → 임베딩 → 벡터 DB (1장)
- 흐름 다이어그램: CSV → 전처리 → 11개 피처 문장화(5단계+수치) → sentence-transformers → ChromaDB + BM25 인덱스
- 임베딩 텍스트 예시 박스
- 하이브리드 검색 구조: 벡터(60%) + BM25(40%)

### 8. 라이브 데모 / 주요 기능 (2장)
- 8a: 감정 기반 ("우울해") + 상황 맞춤 ("카페 독서")
  - [캡처 플레이스홀더 2개]
- 8b: 유사곡 ("Yellow 같은 곡") + 에이전트 로그 + 감정 변화 그래프
  - [캡처 플레이스홀더 2개]

### 9. 토큰 최적화 (1장)
- AS-IS → TO-BE 2행 구성
- Sliding Window: 전체→4턴(8메시지)
- 자동 요약: 5턴마다 LLM 요약 (~90% 절약)
- Care 조건부 실행: 800자 기준

### 10. 응답 지연속도 최적화 (1장)
- AS-IS → TO-BE 3행 구성
- Gate 키워드 분류(LLM 생략), Spotify 병렬 실행, 배치 임베딩(512개), Reranker 배치 조회
- intent별 진행 메시지 [캡처 플레이스홀더]

### 11. 주요 Troubleshooting (1장)
- AS-IS → TO-BE 카드 4개
- 임베딩 품질(2→11피처), 벡터→하이브리드(BM25), popularity 동적 가중치, Care Agent 빈 응답

### 12. 안전장치 (1장)
- 좌: 입력(위기 감지 9개 키워드, 상담 연결) + 출력(금지 표현 6종, Self-Verification)
- 우: [캡처 플레이스홀더] 위기 감지 응답

### 13. 품질 및 성능 평가 (1장)
- 평가 지표 표: 할루시네이션/Intent정확도/Spotify매칭/Latency/가드레일통과/감정추정
- Edge Case 8개
- pytest 테스트 코드

### 14. 한계점 + 확장 방향 + 감사합니다 (1장)
- 좌: 한계점 4개 (정직하게)
- 우: 확장 4개 (PHR/ACTIC/MCP/A/B)
- 하단: "감사합니다" + 기술 스택 한줄
